# README-taskagent

## 1. 项目概述（精简版）

本项目面向养老机构多角色协同照护场景，基于 `LangGraph + Qwen + 工具增强` 构建了可追溯的 Task Agent 多智能体系统；核心创新在于将“长者健康智能计算图”中的老人主体、近况摘要、风险洞察、照护洞察、个性化健康档案、体征洞察、用药洞察等高质量信息设计为可组合工具，供智能体在任务执行过程中自主调用，实现“历史证据可计算、决策过程可解释、报告输出可审计”，并通过 `use_tools=true/false` 对照实验量化验证方案质量提升。  

**计算医学分期创新（A0/A15）**：从 `DataSource` 队列病例提取老年健康生物标志物（Excel），经 **本地序贯疾病分期模型** 与 **贝叶斯后验融合**（全程本地运行）后，由 `tool_疾病分期洞察`、`tool_贝叶斯风险后验` 驱动 **超级 GP（A3）** 及 A6/A8 的分层照护与升级阈值决策。详见 `staging/`、`run_staging_pipeline.py` 与 `/srv/supercare/比赛文档/6_创新点_本地序贯分期与贝叶斯风险融合.md`。  

## 2. 输出规则

- 报告统一 PDF，正文不限制字数；无依据字段留空，不编造。  
- 思维链 `<思考>` 单独展示（日志与PDF中保留），不计入正文字数。  
- D30报告除 PDF 外，额外输出 `a13_D30高价值语料.jsonl`（20行，<=5000字）。  
- 输出命名统一：`a{数字}_{文件名}.ext`。  
  - 示例：`a11_返院一周综合报告.pdf`  
  - 多文件：`a9_护士评估结论与调整建议.pdf`、`a9_照护员周重点任务.pdf`  
- **照护科学实践图谱**：全量测试通过后，自动根据 `output/` 下 **A1-A14 的 PDF** 抽取正文，凝练为 **照护科学行业** 可积累的结构化知识（**不以个案照料为中心叙事**）；图谱以 **超级照护员（A1）、超级护士（A2）、超级全科医师-GP（A3）** 为三认知核心向外辐射，输出：  
  - `output/照护科学实践图谱.json`（节点、边、摘要，便于科研二次分析或导入图库）  
  - `output/照护科学实践图谱.jpg`（**分层 shell 布局**、节点内 **中文换行完整显示**、边关系可读）  
  - 首次绘图会尝试将 **思源黑体 CN** 下载到 `fonts/` 目录（见 `fonts/.gitignore`）  
  - 亦可单独执行：`python /srv/supercare/task-agent/generate_practice_knowledge_graph.py`  
  - 若大模型未返回可解析 JSON，将自动使用「实践链路保底结构」以保证 JPG/JSON 仍可交付。

### 3.4 全周期工作流设计（D-2 到 D30）

Task Agent 的业务工作流面向 BPSD 长者返院后的连续照护闭环，以“自动化智能体执行 + 三类角色中枢协同 + 人工关键确认”为核心机制。流程覆盖返院前接收、返院适应、事件升级、护士评估、远程会诊、中期复盘与 D30 再稳定评估七个阶段。

| 阶段 | 核心目标 | 主要智能体/角色 | 关键输出 |
|---|---|---|---|
| D-2 返院前接收 | 将医院出院信息转化为机构可执行任务 | A4、A5、A6、A2、A3 | 出院摘要结构化包、ECR 状态对比报告、返院适应期专项任务包 |
| D0/D2 返院适应与事件升级 | 对 BPSD 事件进行结构化记录、分级和处置建议生成 | A7、A8、A1、A2 | BPSD 事件报告、BPSD 事件处理建议、照护员现场处置要点 |
| D5 护士评估与任务重配 | 根据适应期执行情况调整护理计划与周重点 | A9、A1、A2 | 护士评估结论与调整建议、照护员周重点任务 |
| D7 会诊准备与协同 | 汇总一周数据并形成 GP/精神科会诊依据 | A10、A11、A12、A3 | 三时点功能对比图、返院一周综合报告、远程会诊记录 |
| D14 中期复盘 | 评估阶段性照护成效，修订干预策略 | A13、A14、A2、A3 | 照护成效中期报告、质量审计意见 |
| D30 再稳定评估 | 形成结案建议、复盘机制并沉淀训练语料 | A13、A14 | D30 再稳定评估报告、高价值语料 JSONL、审计评估报告 |

工作流的工程实现采用 `task_agent_workflow.py` 中的 LangGraph `StateGraph` 编排方式，A1-A14 均可作为独立节点运行，也可串联为完整链路。每个节点读取共享状态中的 `graph_path`、`config_path`、`use_tools` 与当前任务输入；节点输出会回写到状态字典，后续节点可引用前序报告、PDF 路径与关键结论。阶段一 D-2 快速工作流示例见 `快速开始-阶段一D2工作流.md`，其中 A4->A5->A6 后并行分发至 A2/A3，体现“结构化解析、基线对比、任务包生成、护士/GP 确认”的阶段闭环。

### 3.5 照护科学实践图谱生成机制

系统在 A1-A14 报告全部生成后，会进一步执行 `generate_practice_knowledge_graph.py`，从 `output/` 下的 PDF 报告中抽取正文内容，生成面向行业复用的“照护科学实践图谱”。该图谱不是对单一病例的流水账复述，而是将多智能体报告中的可迁移方法论、风险阈值、照护任务、协同分工和成效证据抽象为结构化节点与关系。

图谱生成包含四个步骤：

1. **报告正文抽取**：读取 A1-A14 的 PDF 产物，清洗页眉页脚、思维链标签和重复字段。
2. **知识命题凝练**：由大模型将报告内容抽象为“核心智能体、文档产出、行业知识命题、风险控制点、质控闭环”等节点。
3. **关系构建与审计**：生成“支撑、触发、协同、产出、评估”等关系，并保留来源报告与摘要证据，便于复核。
4. **双格式交付**：输出 `照护科学实践图谱.json` 供科研分析或图库导入，同时输出 `照护科学实践图谱.jpg` 供路演和评审展示。

当前图谱以超级照护员（A1）、超级护士（A2）、超级全科医师-GP（A3）为三大认知核心，向外连接非药物干预个性化设计、跌倒风险环境-行为双轨防控、BPSD 升级阈值、家属焦虑结构化沟通、D30 再稳定评估等知识节点，体现“数据证据 -> 多角色协同 -> 任务执行 -> 审计评估 -> 语料沉淀”的闭环创新。

## 4. LangGraph 集成示例

```python
from langgraph.graph import START, END, StateGraph
from typing import TypedDict, Dict, Any
from agent_a8_escalation_decision_agent import get_langgraph_node

class State(TypedDict, total=False):
    graph_path: str
    # 图谱文件绝对路径，类型：str。
    # 作用：指定“长者健康智能计算图”JSON数据源。
    # 建议：传 /srv/supercare/data/xxx.json，未传时由Agent内部使用默认路径。

    config_path: str
    # 模型配置文件路径，类型：str。
    # 作用：读取Qwen的api_key/base_url/model/max_tokens等参数。
    # 建议：使用 /srv/supercare/config/data_agent1_config.json。

    use_tools: bool
    # 是否启用工具抽取，类型：bool。
    # True：先调用14类工具抽取历史证据，再交给LLM；False：仅基于用户输入（用于A/B对照测试）。

    a8_input: str
    # A8节点输入文本，类型：str。
    # 作用：放“BPSD事件报告内容”或“待升级判断的问题描述”。
    # 建议：写成结构化长文本（时间、场景、行为、处置、生命体征、家属状态）。

    a8_output: Dict[str, Any]
    # A8节点输出对象，类型：Dict[str, Any]。
    # 常见字段：
    # - success: bool，执行是否成功
    # - text: str，模型正文输出
    # - thinking: str，<思考>内容（若有）
    # - pdf_path: str，生成的PDF路径（若该Agent输出PDF）
    # - error: str，异常信息（失败时）

builder = StateGraph(State)
# StateGraph(State)：用上面的State定义工作流“共享状态结构”。

builder.add_node("a8", get_langgraph_node("a8_input", "a8_output"))
# add_node(节点名, 节点函数)：
# - "a8" 是工作流中的节点标识（可自定义但需唯一）
# - get_langgraph_node("a8_input","a8_output") 表示：
#   从state["a8_input"]读取输入，执行A8逻辑后写入state["a8_output"]

builder.add_edge(START, "a8")
# add_edge(START, "a8")：定义起始边，工作流启动后先执行a8节点。

builder.add_edge("a8", END)
# add_edge("a8", END)：定义结束边，a8执行完成后结束流程。

app = builder.compile()
# compile()：编译为可运行应用，可通过 app.invoke(initial_state) 执行。
```
